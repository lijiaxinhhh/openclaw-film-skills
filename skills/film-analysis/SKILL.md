# Film Analysis - 拉片分析工具

一站式电影/纪录片拉片分析工具，整合剧本分析、镜头分析、场景时间轴、音乐音响分析、剪辑节奏分析，支持从视频链接或本地文件完成完整拉片。

## 核心特性

- **完整拉片流程**：从视频获取到综合报告的全流程自动化
- **剧本分析**：逐字稿生成、说话人区分、故事结构拆解、角色弧线追踪
- **镜头分析**：景别识别、运镜分析、构图解读、色彩光线评估、五维评分
- **场景时间轴**：场次排序、场景功能标注、转场分析
- **音乐音响分析**：配乐情绪、音效层次、声画关系
- **剪辑节奏分析**：镜头时长分布、剪辑手法识别、Walter Murch 六法则
- **纪录片专项**：叙事视角、真实素材 vs 搬演、采访构图
- **多源支持**：B站、YouTube、抖音、小红书及本地文件
- **智能降级**：字幕提取四级降级策略（API → 内嵌字幕 → OCR → ASR）

## 依赖工具

| 工具 | 用途 |
|------|------|
| **ffmpeg** | 视频/音频处理、帧提取、格式转换 |
| **yt-dlp** | 视频下载（B站/YouTube/抖音/小红书） |
| **PySceneDetect** | 场景自动检测与切分（`detect-content`） |
| **Whisper / mlx-whisper** | 语音转文字（逐字稿生成） |
| **RapidOCR** | 图片文字识别（过场字幕、内嵌字幕提取） |
| **FunASR** | 语音识别（ASR 降级方案） |
| **Vision 模型** | 图像分析（镜头分析、构图解读、色彩光线） |

> 安装检查：运行前确保上述工具已安装。可执行 `which ffmpeg yt-dlp` 等命令验证。

## 工作流程

### ⚠️ 翻译原则

**始终从音频重新翻译，不使用现有字幕的翻译。** 原因：
- 现有字幕翻译常不准确，不够信达雅
- 翻译不符合故事情景和语境
- 官方字幕可能有删减或改编

**流程：**
1. 用 Whisper 从音频提取原始语言逐字稿
2. 由 LLM 结合上下文、故事情景进行意译（非直译）
3. 专有名词、俚语、文化梗需特别处理
4. 现有字幕仅作参考对照，不作为最终翻译来源

### 第一步：视频获取与预处理

**视频下载**
- 本地文件：直接使用文件路径
- 在线链接：使用 yt-dlp 下载
  - B站：`yt-dlp -o "output.mp4" <bilibili-url>`
  - YouTube：`yt-dlp -o "output.mp4" <youtube-url>`
  - 抖音/小红书：同上，yt-dlp 自动识别平台

**音频提取**
```bash
ffmpeg -i input.mp4 -vn -acodec pcm_s16le -ar 16000 -ac 1 audio.wav
```

**场景检测与自动切分**
```bash
scenedetect -i input.mp4 detect-content -t 27.0 list-scenes save-images
```
- 输出场景列表（时间戳 + 缩略图）
- 阈值 `-t` 可根据影片风格调整（纪录片偏低，动作片偏高）

**智能字幕提取（四级降级策略）**
1. **B站 API**：通过 B站接口获取官方字幕（最准确）
2. **内嵌字幕**：ffmpeg 提取视频内嵌字幕轨（`-map 0:s:0`）
3. **RapidOCR**：对字幕区域截图进行 OCR 识别
4. **FunASR**：音频语音识别（最终降级方案）

### 第二步：剧本分析

**逐字稿生成**
- 使用 Whisper / mlx-whisper 对音频进行语音转文字
- 输出带时间戳的逐字稿（SRT/JSON 格式）
- 默认翻译成中英文对照版本

**逐字稿衍生产出**
在逐字稿基础上完成：
- 分镜头剪辑剧本（包含镜头编号、时间码、画面描述、对白、音效/音乐）
- 文学剧本要像专业电影剧本一样详细，每场戏必须包含：
- **场景标题**：场景编号、内/外景、日/夜、地点
- **时间**：具体时间（如 "1972年1月5日" 或 "清晨"）
- **地点**：具体地点（如 "纽约公寓"、“巴黎街头”）
- **人物**：出场人物列表
- **故事内容**：详细的叙事性描述，包括动作、表情、环境
- **对白**：中英对照，标注说话人和说话对象
- **音效/音乐**：环境音、配乐、BGM
- **剧作分析**：每场戏后面的剧作分析（重要场次必须有）
  - 场景功能（建置/转折/高潮）
  - 人物弧线变化
  - 对白潜台词
  - 主题表达
  - 情感变化

**逐字稿类型标记**
逐字稿中必须区分以下类型并明确标记：
- **旁白**（Narration）：画外音叙述，标记为 `[旁白]`，注明叙述人（如 Rachel、Sheila 等）
- **对话**（Dialogue）：现场采访或对话，标记为 `[对话]`，注明说话人和说话对象
- **BGM歌词**（Music Lyrics）：背景音乐中的歌词，标记为 `[BGM]`，注明歌曲名和艺术家
- **电话录音**（Phone Call）：电话通话内容，标记为 `[电话]`，注明通话双方
- **档案录音**（Archive Audio）：历史录音资料，标记为 `[档案]`，注明录音来源和时间

每段逐字稿必须标注清楚：
1. 说话类型（旁白/对话/BGM/电话/档案）
2. 说话人是谁
3. 说话对象是谁（如果是对话）
4. 说话场景（如果能判断）

**说话人区分**
- 基于音频特征识别不同说话人
- 标注每段对白的说话人身份

**旁白 vs 采访识别**
- 区分旁白叙述（off-screen）与现场采访（on-screen）
- 标注叙述视角

**故事大纲提炼**
故事大纲要尽可能详细丰富，包括：
- 完整的三幕结构分析（建置/发展/高潮/解决）
- 每一幕的详细内容描述
- 关键转折点标注
- 主题线索追踪
- 情感曲线变化

**人物关系与视角**
故事大纲必须涵盖以下人物视角和关系线：
- **Rachel 作为女儿的视角**：对母亲的思念、对父亲的理解、对家庭关系的反思
- **Rachel 作为女友/妻子的视角**：与 Joseph 的亲密关系、对婚姻的态度、对生育的恐惧与渴望
- **Rachel 与父亲的关系**：父亲的沉默、父女之间的隔阂与理解
- **Rachel 与自我的关系**：作为摄影师的身份认同、对过去的执念与放下
- **亲密关系的探讨**：Rachel 如何通过寻找母亲来理解自己的亲密关系模式

**背景研究**
- 电影/纪录片背景介绍
- 主人公真实经历（纪录片适用）
- 创作背景与制作语境

**剧本结构拆解**
- **Act（幕）**：三幕结构或其他宏观结构
- **Sequence（序列）**：幕内的情节单元
- **Scene（场景）**：单一时空内的动作
- **Beat（节拍）**：场景内最小的戏剧单元

**深度剧作分析**
剧作分析不再作为独立模块，而是整合到文学剧本中：
- 每场戏后面附上剧作分析
- 重要场次必须有详细分析
- 分析内容包括：
  - 叙事结构的精妙之处
  - 人物塑造手法
  - 对白写作技巧（潜台词、冲突、信息量）
  - 场景转换技巧
  - 主题表达方式
  - 情感转折点
  - 角色关系变化

**角色弧线追踪**
- 主要角色的性格变化轨迹
- 关键转折点与动因分析
- 角色关系演变

**对白分析**
- 潜台词：角色真正想表达的内容
- 冲突：对白中的戏剧张力
- 信息量：每句对白传递的故事信息

**场景功能标注**
- 建置（Setup）：建立世界观与角色
- 转折（Turning Point）：推动情节变化
- 高潮（Climax）：矛盾最激烈的时刻
- 解决（Resolution）：冲突的化解
- 过渡（Transition）：连接不同段落

**叙事技巧识别**
- 悬念与伏笔
- 闪回与闪前
- 多线叙事与交叉剪辑
- 不可靠叙述者
- 元叙事与打破第四面墙

**可选 Prescene API 增强**
- 调用 Prescene API 获取专业剧本分析数据
- 与本地分析结果交叉验证

### 第三步：镜头分析

**过场字幕识别**
- 黑底白字等过场文字的 OCR 识别
- 提取时间、地点等叙事信息

**逐镜头分析策略**
- **固定镜头**：逐镜头分析（一个镜头一个分析）
- **不稳定镜头**：逐帧分析（连续动作需要更细粒度）

**时间轴与截图要求**
每个镜头分析点必须附带：
- 原片时间轴节点（MM:SS 格式）
- 对应场景的截图引用（用相对路径）

**景别识别**
| 景别 | 英文 | 描述 |
|------|------|------|
| 大远景 | Extreme Long Shot | 展示宏大环境 |
| 远景 | Long Shot | 人物在环境中较小 |
| 全景 | Full Shot | 人物全身可见 |
| 中景 | Medium Shot | 膝盖以上 |
| 中近景 | Medium Close-up | 胸部以上 |
| 近景 | Close-up | 肩部以上，面部清晰 |
| 特写 | Extreme Close-up | 面部局部或细节 |
| 大特写 | Macro / Insert | 极端细节 |

**运镜分析**
- **推（Push In）**：镜头向前移动，聚焦主体
- **拉（Pull Out）**：镜头向后移动，展示环境
- **摇（Pan）**：镜头水平旋转
- **移（Dolly / Tracking）**：镜头整体移动
- **跟（Follow）**：跟随主体移动
- **升（Crane Up）**：镜头垂直上升
- **降（Crane Down）**：镜头垂直下降
- **手持（Handheld）**：制造紧张/纪实感
- **斯坦尼康（Steadicam）**：流畅移动

**构图解读**
- 三分法（Rule of Thirds）
- 对称构图（Symmetry）
- 引导线（Leading Lines）
- 景深（Depth of Field）：浅景深聚焦 vs 深景深全景
- 框中框（Frame within Frame）
- 负空间（Negative Space）

**色彩与光线分析**
- 色调（暖/冷/中性）
- 色彩饱和度
- 光线方向（正面/侧面/逆光/顶光）
- 光线质感（硬光/柔光）
- 明暗对比与情绪关系

**五维评分系统**
| 维度 | 说明 |
|------|------|
| 🎨 美感 | 画面的视觉美感与艺术性 |
| 🔍 可信度 | 画面的真实感与说服力 |
| 💥 冲击力 | 视觉冲击与情感震撼 |
| 🧠 记忆度 | 画面的辨识度与难忘程度 |
| 🎭 趣味度 | 画面的趣味性与观赏性 |

**动态权重模式**
- **Hook 模式**：冲击力 ×2，记忆度 ×1.5（开场/预告片）
- **叙事模式**：可信度 ×2，美感 ×1.5（剧情推进）
- **氛围模式**：美感 ×2，趣味度 ×1.5（情绪渲染）
- **商业模式**：冲击力 ×2，趣味度 ×2（广告/宣传片）

**筛选等级**
- ⭐ **MUST KEEP**：不可或缺的镜头，核心画面
- ✅ **USABLE**：可用镜头，辅助叙事
- ❌ **DISCARD**：建议舍弃的镜头

### 第四步：场景时间轴（内部数据结构，非独立输出）

> 场景时间轴作为内部数据结构，不单独作为报告输出模块。场景数据整合进镜头语言分析和其他模块中。

**内部场景数据**
- 按时间戳排列所有场景
- 标注每场的起止时间
- 场景内容概要、参与角色、场景功能
- 按序列/幕进行段落级总结
- 识别节奏变化点

**转场类型与意图分析**（整合进镜头语言分析）
- 硬切（Cut）：直接切换
- 叠化（Dissolve）：时间流逝/梦境
- 淡入淡出（Fade）：段落结束/开始
- 划变（Wipe）：场景并列
- 跳切（Jump Cut）：时间跳跃/紧张感
- 匹配剪辑（Match Cut）：视觉/概念连接
- 声桥（Sound Bridge）：声音先行/延后

### 第五步：音乐音响分析

**音乐识别与出处**
- 识别并列出片中使用的音乐作品（曲名、作曲家/艺术家）
- 找到音乐出处并附上链接（Spotify/YouTube/Apple Music 等）
- 每段音乐标注在原片中的时间轴节点（MM:SS）
- 分析音乐与画面的情绪关联

**配乐情绪分析**
- 音乐与画面情绪的对应关系
- 音乐对叙事的推动作用
- 主题音乐的变奏与发展

**音效层次分析**
- 前景音效：直接叙事相关的声音
- 中景音效：环境中的具体声音
- 背景音效：氛围底噪

**环境音与静默分析**
- 环境音的叙事功能
- 静默的戏剧效果
- 声音的象征意义

**声画关系**
- 同步（Sync）：声音与画面一致
- 对位（Counterpoint）：声音与画面形成对比
- 声画分离（Split）：声音独立于画面叙事

### 第六步：剪辑节奏分析

**镜头时长分布**
- 统计各场景/段落的平均镜头时长
- 识别长镜头与快剪段落

**剪辑加速/减速点**
- 加速：节奏加快，制造紧张/兴奋
- 减速：节奏放慢，制造沉思/情感

**特殊剪辑手法**
- 跳切（Jump Cut）：同一主体的不连续剪辑
- 匹配剪辑（Match Cut）：视觉元素的连接
- 交叉剪辑（Cross Cut）：两条线索交替
- 平行剪辑（Parallel Edit）：同时发生的事件
- 动作剪辑（Action Cut）：在动作中切换

**Walter Murch 剪辑六法则**
1. **情感（Emotion）**：51% — 剪辑点是否传达正确情感
2. **故事（Story）**：23% — 是否推进故事
3. **节奏（Rhythm）**：10% — 是否符合节奏感
4. **视线（Eye-trace）**：7% — 观众视线是否顺畅
5. **二维屏幕方向（Screen Direction）**：5% — 轴线是否正确
6. **三维空间连续性（3D Space）**：4% — 空间关系是否连贯

### 第七步：纪录片专项（如适用）

**叙事视角分析**
- 全知视角：上帝视角，无所不知
- 限制视角：跟随特定人物
- 主观视角：第一人称叙述
- 观察式：直接电影/真实电影风格

**真实素材 vs 搬演**
- 真实影像：原始记录素材
- 搬演/重现：基于真实事件的演绎
- 动画/图示：辅助说明的视觉元素
- 档案素材：历史影像资料

**采访构图分析**
- 采访角度：正面/侧面/倾斜
- 视线方向：直视镜头 vs 看向画外
- 背景选择：环境采访 vs 纯色背景
- 光线处理：自然光 vs 人工布光

**纪录片分析时间轴与截图要求**
每个分析点必须附带：
- 原片时间轴节点（MM:SS 格式）
- 对应场景的截图引用

### 第八步：综合报告输出

**完整拉片报告（Markdown）**
- 包含以上所有分析维度的整合报告
- 时间轴 + 剧本 + 镜头 + 音乐 + 剪辑的多维度交叉分析
- 关键发现与创作洞察

**与 video-expert-analyzer 交叉引用**
- 调用 video-expert-analyzer 进行镜头评分与筛选
- 将评分结果整合进拉片报告
- 提供 MUST KEEP / USABLE / DISCARD 的完整镜头清单

## 使用示例

### 本地视频文件分析

```
请对 /path/to/movie.mp4 进行完整拉片分析
```

工具将自动：
1. 提取音频并进行场景检测
2. 生成逐字稿与剧本分析
